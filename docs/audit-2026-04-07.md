## 1. Security Issues

### CRITICAL

| # | Issue | File | Description | Status |
|---|-------|------|-------------|--------|
| C-1 | OAuth user/org ID spoofing | `src/app/api/v1/oauth/authorize/route.ts` | Authorize endpoint accepts `user_id` and `org_id` from request body but never verifies they match the authenticated session. Any logged-in user can generate OAuth tokens for any other user/org. | [ ] |
| C-2 | Account deletion token in plaintext | `src/app/api/v1/account/delete/route.ts` | Deletion tokens stored unhashed. DB breach lets attacker delete any account. Should be SHA-256 hashed like API keys. | [ ] |
| C-3 | Email action tokens in plaintext | `src/lib/notifications/tokens.ts` | Approve/reject email tokens stored unhashed. DB breach exposes ability to approve/reject any pending request. | [ ] |

### HIGH

| # | Issue | File | Description | Status |
|---|-------|------|-------------|--------|
| H-1 | Client secret logged to console | `src/app/api/v1/oauth/token/route.ts:293` | Logs first 12 chars of client_secret on failure | [ ] |
| H-2 | No rate limiting on signup | `src/app/api/auth/signup/route.ts` | Enables mass account creation and email abuse | [ ] |
| H-3 | No rate limiting on account deletion | `src/app/api/v1/account/delete/route.ts` | Attacker with session could spam deletion emails | [ ] |
| H-4 | Tracking endpoint unauthenticated + unthrottled | `src/app/api/tracking/visit/route.ts` | Can fill DB with junk data | [ ] |
| H-5 | SSRF bypass via DNS rebinding | `src/lib/api/ssrf.ts` | Checks hostname string but doesn't resolve DNS. Attacker domain resolving to `169.254.169.254` bypasses protections | [ ] |
| H-6 | Email actions use GET for state changes | `src/app/api/email-actions/[token]/route.ts` | Email client link previews can auto-approve/reject | [ ] |
| H-7 | CSP allows unsafe-eval + unsafe-inline | `next.config.ts:36` | Effectively disables XSS protection | [ ] |

### Positive Security Findings (Already Good)
- API keys hashed with SHA-256
- OAuth tokens hashed, refresh token rotation with reuse detection
- PKCE support for OAuth
- Org-scoped data access via RLS
- Webhook signature verification (Slack, Discord, Teams, Telegram)
- HTML sanitization via DOMPurify
- Zod validation on all endpoints
- Audit logging on most operations
- Security headers (HSTS, X-Frame-Options, etc.)
- Stripe webhook signature verification
- MFA enforcement
- No raw SQL
- Key rotation grace periods
- Column allowlists (never returns `api_key_hash`)
- .env files gitignored

## 4. Competitive Landscape

### Direct Competitors

**Zapier Human in the Loop** - Built-in feature that pauses Zaps for approval. Limited to linear if/then logic, no conditional routing, no SLA, no escalation. Pricing scales badly ($500+/mo for content teams).

**n8n HITL** - Uses Wait nodes + notification channels. More flexible than Zapier but requires building your own approval UI.

**Make** - Basic approval checkpoints but no dedicated approval UX.

### Adjacent Competitors

| Product | Focus | Pricing |
|---------|-------|---------|
| Workato | Enterprise iPaaS with governance, RBAC, audit logs | $50K+/year platform fee |
| Tray.io | Flexible iPaaS for technical teams | Usage-based, enterprise |
| Retool | Internal tool builder with approval workflows (beta) | Per-user |
| ApproveIt | Slack-native approval workflows | SaaS |
| Cflow | No-code approval workflow automation | Mid-range |
| Vendr | SaaS procurement approvals | Enterprise |

### User Pain Points (from reviews/forums)

1. Multi-stakeholder routing is broken in linear tools
2. Pricing scales badly at high volume
3. Debugging multi-step workflows is painful
4. Approvers see raw data, not formatted context
5. Approvals stall with no auto-escalation

### OKRunit Advantages
Already has: SLA tracking, escalation policies, delegation, risk scoring, trust engine, multi-channel notifications (Slack/Teams/Discord/Telegram/email), audit logging, dedicated approval UX. These are significant differentiators.

### Opportunities to Differentiate Further
1. Approval templates
2. Conditional approval routing (surface the rules engine better)
3. Rich approval context (formatted previews)
4. Calendar-based auto-delegation (unique in market)
5. Approval analytics
6. Mobile push for one-tap approve/reject
7. Batch approvals with guardrails
